Build a full-stack web application called **SmartConvert**.

The application should have a modern, responsive UI with React (Vite + TypeScript + Tailwind CSS) and a FastAPI backend written in Python.

## Features

### 1. File Conversion

Implement the following conversions:

* DOCX → PDF
* PDF → DOCX
* PDF → TXT
* PDF → Markdown
* Images (PNG/JPG) → PDF
* PDF → Images

All conversions should preserve formatting as much as possible.

---

### 2. File Compression

Allow users to upload one or multiple files.

Provide a dropdown to choose one of the following algorithms:

* ZIP (DEFLATE)
* Gzip
* Bzip2
* LZMA
* Brotli
* Zstandard
* LZ4
* Snappy

Implement each compression algorithm using the appropriate Python library.

After compression, display:

* Original size
* Compressed size
* Compression ratio
* Compression time
* Decompression time
* Compression speed
* Algorithm used

Allow users to download the compressed file.

---

### 3. Adaptive Compression

Implement an Adaptive Compression option.

This should not use machine learning initially.

Instead:

* Analyze uploaded files.
* Detect MIME type.
* Detect text vs binary.
* Estimate entropy.
* Detect already compressed formats.
* Estimate repetition.
* Estimate average line length for text.

Choose the most appropriate algorithm using heuristics.

Provide a "Why this algorithm?" panel explaining every decision.

Example:

Selected Algorithm:
Zstandard

Reason:

* Text file detected
* Moderate entropy
* Large repeated sequences
* Fast decompression preferred

---

### 4. Benchmark Mode

Allow users to select multiple algorithms simultaneously.

Run every selected algorithm.

Display results in a sortable comparison table showing:

* Algorithm
* Compression ratio
* Compression time
* Decompression time
* Memory usage (if measurable)
* Final size

Highlight the best performer.

---
